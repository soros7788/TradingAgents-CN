## 工作流步骤

### 步骤1: 接收截图数据
- 用户上传同花顺/华宝证券截图
- 读取截图提取: 总资产、可用资金、持仓明细(名称/代码/数量/成本/现价/市值/盈亏)
- 如果用户没有截图, 跳过此步骤, 使用Excel已有数据

### 步骤2: 更新Excel
- 更新账户总表(总资产/可用资金/日期)
- 更新持仓表现价
- 重算公式(recalc.py)
- 验证零错误

### 步骤3: 合规核查
- 检查每只持仓: 止损价 vs 现价
- 检查WAIVED豁免状态
- 检查仓位占比(单股<=35%, 总仓位<=70%)
- 检查有效持仓数
- 输出合规报告

### 步骤4: 候选扫描（全市场）→ 自动写入候选池
当用户说"候选扫描"时，一键执行:
1. **沪A全量**: 新浪分页接口获取全部沪A主板(~984只, 排除688科创板)
2. **深市全量**: 生成000主板+002中小板代码(1998个) → 新浪批量获取现价(50只/批) → 过滤ST和停牌(~1255只有效)
3. **背驰分析**: 逐只跑日线分析, 总覆盖~2239只, 耗时~95秒
4. **候选池写入**: 沪深均衡分配(各取top15), 写入Excel候选池, 重算公式

**排除板块**: 3开头创业板 + 688科创板 + 北交所

**高价股纳入**: **不排除高价股**。资金不足时提示"需转入X元"
- 资金转入机制: 转入→买入1手→清仓后转出
- 按账户规模执行，不影响长期仓位管理

**确认标准**: ratio<60% + DL_P>0.8 + valid=True（三条件缺一不可）

**价格异常兜底**: 日K收盘价与市场现价偏差>10倍时, 使用市场现价(修复深市复权问题)

**候选池排序**: score降序 + ratio升序(同分时背驰越强的优先)

### 步骤5: 心态日志提醒
- 每次工作流结束时, 提醒用户填写心态日志
- 注意: 截图录入属于工作流, 不计入"非工作流看盘次数"

## 命令
```bash
python daily_workflow.py compliance   # 止损+仓位合规核查
python daily_workflow.py scan          # 日线全市场扫描 + 写入候选池 + 重算公式
python daily_workflow.py intraday      # 盘中30min扫描候选池 + 5min确认 + 持仓止损检查
python daily_workflow.py account        # 账户总览
python daily_workflow.py holdings       # 持仓明细
```

## 盘中扫描策略（多级别）
| 级别 | 周期 | 扫描范围 | 耗时 | 触发方式 |
|---|---|---|---|---|
| 日线 | 1天 | 全市场2238只 | ~95s | 盘后/手动 scan |
| 30min | 30分钟 | 候选池27只 | ~1s | 定时 intraday |
| 5min | 5分钟 | 30min确认标的 | ~0.4s/只 | 30min确认后自动 |
| 1min | 1分钟 | 持仓标的 | ~0.2s/只 | 止损告警时手动 |

**intraday 命令执行流程:**
1. 读取候选池 → 30min级别逐只扫描(~1秒)
2. 30min有确认 → 5min级别精确买点
3. 持仓止损检查 → 破止损立即告警

## 高价股资金转入/转出机制
- 当确认信号标的价格 > 可用资金/100 时，提示需转入金额
- 转入金额 = 现价 * 100 - 可用资金
- 买入后按正常止损规则执行
- 清仓后多余资金转出账户外
